# Фаза 1: Ядро фреймворка (MVP)

Данный документ описывает реализацию базового ядра фреймворка MiniNG.

## 1. Декоратор `@Component`
Файл: `src/core/decorators/component.ts`

Реализует Angular-подобный декоратор классов, который сохраняет метаданные компонента (селектор и HTML-шаблон) с помощью `reflect-metadata`.
```typescript
export interface ComponentConfig {
  selector: string;
  template: string;
}
```

## 2. Template Parser
Файл: `src/core/renderer/template-parser.ts`

Отвечает за парсинг HTML-шаблона, привязанного к компоненту.
Функции парсера:
- **Интерполяция `{{ }}`**: обходит текстовые узлы DOM с помощью `document.createTreeWalker`, находит выражения в двойных фигурных скобках и заменяет их на значения свойств из инстанса компонента.
- **Привязка событий `(event)`**: ищет атрибуты в формате `(eventName)="methodName()"`, извлекает имя события и метода, после чего вызывает `addEventListener`, связывая его с соответствующим методом экземпляра.

## 3. Renderer
Файл: `src/core/renderer/renderer.ts`

Связующее звено между классом компонента и DOM-деревом:
1. Читает конфигурацию компонента через функцию `getComponentConfig`.
2. Создает инстанс компонента (пока напрямую, без Dependency Injection).
3. Парсит шаблон с помощью `TemplateParser`.
4. Монтирует полученный `DocumentFragment` в хост-элемент на странице.

## 4. Bootstrap
Файл: `src/core/bootstrap.ts`

Точка входа во фреймворк. Функция `bootstrap(ComponentClass)`:
- Ищет на странице элемент по `selector` переданного компонента.
- Если элемент найден, вызывает `Renderer.render`.

## Итог
Реализован базовый функционал создания UI-компонентов с однонаправленным рендерингом данных и обработкой пользовательских событий (кликов).
