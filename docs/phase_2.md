# Фаза 2: Реактивность (Change Detection)

В данной фазе реализовано автоматическое обновление DOM при изменении свойств компонента.

## 1. Реактивный Proxy
Файл: `src/core/renderer/change-detection.ts`

Функция `createReactiveProxy` оборачивает исходный экземпляр компонента в объект `Proxy`.
При любом изменении свойства (перехват `set`):
1. Значение обновляется в исходном объекте.
2. В очередь микрозадач (`queueMicrotask`) помещается функция обновления `onUpdate`.
3. Использование флага `pendingUpdate` позволяет объединять несколько синхронных мутаций в один цикл перерисовки (batching).

## 2. Обновление привязок (Bindings)
Файл: `src/core/renderer/template-parser.ts`

В парсер шаблонов добавлен статический метод `updateBindings`:
- Группирует все привязки по текстовым узлам (DOM `Node`), чтобы обновлять каждый узел только один раз за цикл.
- Подставляет актуальные значения переменных из экземпляра компонента.
- Сравнивает новое значение узла с текущим, и если оно изменилось — обновляет `node.nodeValue`.

## 3. Интеграция в Renderer
Файл: `src/core/renderer/renderer.ts`

Теперь `Renderer` не просто отдаёт созданный класс, а:
- Оборачивает экземпляр в `createReactiveProxy`.
- Передает `TemplateParser.updateBindings` в качестве коллбэка, который будет срабатывать при мутациях.
- Парсинг шаблона происходит с использованием созданного прокси, что гарантирует правильный контекст (`this`) в обработчиках событий.

## Итог
Теперь разработчику достаточно просто изменить свойство класса (например, `this.count++`), и интерфейс автоматически обновится без необходимости вручную вызывать функции перерисовки.
