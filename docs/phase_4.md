# Фаза 4: Расширения v2.0 (Жизненный цикл, директивы, привязки)

В этой фазе мы превратили простой шаблонизатор в более зрелый фреймворк, добавив хуки жизненного цикла и структурные изменения DOM.

## 1. Жизненный цикл `OnInit`
Файл: `src/core/lifecycle/hooks.ts`

Был добавлен интерфейс `OnInit`. В ядре рендеринга (`Renderer.render`) теперь проверяется наличие метода `ngOnInit` у инстанса компонента. 
Если метод существует, он выполняется в конце цикла синхронного монтирования (через `queueMicrotask`), что позволяет инициализировать состояние компонента тогда, когда зависимости внедрены.

## 2. Структурная директива `*ngIf`
Файл: `src/core/renderer/template-parser.ts`

Реализован мощный механизм манипуляции DOM. 
Алгоритм обработки:
1. При нахождении элемента с `*ngIf="expr"`, парсер вырезает его из исходного шаблона, а на его место устанавливает невидимый маркер в виде HTML-комментария (`<!-- ngIf: expr -->`).
2. Шаблонный элемент сохраняется в памяти (`cloneNode`).
3. Создается объект `ViewBinding`, который реагирует на изменения `expr`.
4. При каждом обновлении (из-за Change Detection), парсер проверяет truthy-значение. Если оно `true`, оригинальный элемент клонируется, рекурсивно парсится на наличие других директив/биндингов и вставляется в DOM сразу после комментария. Если `false` — удаляется из DOM.

## 3. Property Binding (`[prop]`)
Файл: `src/core/renderer/template-parser.ts`

Появилась возможность динамически привязывать свойства к элементам через квадратные скобки. Парсер ищет атрибуты вида `[attr]="expr"`:
- Для булевых значений (например, `[disabled]="isLocked"`) парсер использует `setAttribute / removeAttribute`, чтобы элемент корректно включался и выключался.
- Для других типов значений он устанавливает прямое свойство DOM-объекта (напр. `el.value = ...`).
